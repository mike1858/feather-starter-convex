# Plan 999.4-02: Cross-Entity Panel Templates + Detail Page Update

## Objective

Create the 4 cross-entity panel Handlebars templates (checklist, timeline, table, cards) and update the detail page template to use the resolver output for rendering related records. After this plan, generating a feature with `detailView.relatedRecords` in its YAML produces a detail page with real, rendered panels.

## Why This Matters

The cross-entity resolver (`templates/cross-entity/resolver.ts`) already produces a typed `DetailPageContext` with `ResolvedRelatedRecord[]`, but there are no templates to render the panels. This is the last gap between YAML declaration and rendered UI for cross-entity views. CalmDo's tasks page (with subtasks as checklist, work-logs as timeline, activity-logs as timeline) is the primary validation target.

## must_haves

- [ ] 4 panel templates exist: checklist, timeline, table, cards
- [ ] Each panel template receives a `ResolvedRelatedRecord` context and renders appropriate UI
- [ ] Detail page template conditionally includes panel components based on `detailView.relatedRecords`
- [ ] Panels support inline editing flag (`hasInlineEditing` from resolver)
- [ ] Panel rendering integrated into the pipeline's scaffold step
- [ ] Tests verify each panel renders correctly for its display type
- [ ] `npm test` passes at 100% coverage

<action>
1. Create `templates/cross-entity/panel-renderer.tsx.hbs`:
   A wrapper component that takes a `DetailPageContext` and renders the appropriate panel for each related record:

   ```tsx
   // Generated panel renderer for {{parentEntity}} detail page
   {{#each relatedRecords}}
   {{#ifEq display "checklist"}}
   import { {{sourcePascal}}ChecklistPanel } from "./{{source}}-checklist-panel";
   {{/ifEq}}
   {{#ifEq display "timeline"}}
   import { {{sourcePascal}}TimelinePanel } from "./{{source}}-timeline-panel";
   {{/ifEq}}
   {{#ifEq display "table"}}
   import { {{sourcePascal}}TablePanel } from "./{{source}}-table-panel";
   {{/ifEq}}
   {{#ifEq display "cards"}}
   import { {{sourcePascal}}CardsPanel } from "./{{source}}-cards-panel";
   {{/ifEq}}
   {{/each}}

   export function {{parentPascal}}Panels({ parentId }: { parentId: Id<"{{parentEntity}}"> }) {
     return (
       <div className="space-y-6">
         {{#each relatedRecords}}
         <{{sourcePascal}}{{pascalCase display}}Panel parentId={parentId} />
         {{/each}}
       </div>
     );
   }
   ```

2. Update `templates/feature/detail-page.tsx.hbs`:
   - Add conditional import: `{{#if detailView.relatedRecords}}import { {{pascalName}}Panels } from "./{{name}}-panels";{{/if}}`
   - Add panel rendering section after the main entity fields:
   ```tsx
   {{#if detailView.relatedRecords}}
   <{{pascalName}}Panels parentId={id} />
   {{/if}}
   ```

3. Update `templates/pipeline/render.ts` and `templates/pipeline/scaffold.ts`:
   - When a feature has `detailView.relatedRecords`, run the cross-entity resolver
   - For each related record, render the appropriate panel template with the resolved context
   - Render the panel-renderer template with the full DetailPageContext
   - Include all panel files in the scaffold output

4. Update `templates/cross-entity/index.ts` to export all panel-related types and the resolver.
</action>
