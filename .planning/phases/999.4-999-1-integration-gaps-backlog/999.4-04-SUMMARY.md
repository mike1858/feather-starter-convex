# Plan 999.4-04: feather update CLI Thorough Testing + Plop Unification -- Summary

Comprehensive integration testing for the feather update CLI, unit tests for feather init, --force flag bug fix, Plop gen:feature wired to use the pipeline internally, and full round-trip verification against 8 CalmDo-migrated features.

## What Was Built

1. **feather update integration tests** (9 scenarios) -- covers regeneration, uncommitted change detection, --force override, --dry-run preview, missing YAML handling, custom code preservation, multi-feature updates, and YAML validation errors. All tests use real temp directories with git operations.

2. **feather init unit tests** (6 scenarios) -- covers directory creation, --name option, basename default, idempotency, feather.yaml preservation, and schema validation.

3. **--force flag bug fix** -- the delete step in update.ts was incorrectly gated behind `!options.force`, preventing --force from cleaning generated directories before regeneration. Fixed to gate only on `!options.dryRun`.

4. **Plop-to-pipeline unification** -- `npm run gen:feature` now delegates to the pipeline's `generateFeature()` after collecting interactive answers. A `gen:feature-from-yaml` generator was added for direct YAML-to-code generation. Legacy Plop actions remain as fallback via `FEATHER_USE_PIPELINE=false`.

5. **Round-trip verification** -- `feather update` successfully regenerated all 8 features (40 files), tests pass at 716/716, and running update twice produces zero diff (idempotent).

## Key Decisions

- Used `FEATHER_USE_PIPELINE` env var (default: `true`) for Plop-to-pipeline toggle, keeping backward compatibility
- Deprecated `generators/utils/wiring.js` with comment pointing to `templates/wiring/` (ts-morph replacement)
- Fixed duplicate `creatorId` in generated schema fragments by filtering it from YAML field entries (it is always appended as a system field)

## Test Results

- 74 test files, 716 tests passed, 100% coverage
- feather update: 8 features, 40 files regenerated
- Idempotency verified: second update produces zero git diff
