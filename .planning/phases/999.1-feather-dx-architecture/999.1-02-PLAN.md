# Plan 999.1-02: ts-morph AST Wiring Layer — Reliable File Modification

## Objective

Build an AST-based wiring layer using ts-morph that replaces the current manual/template-based approach to modifying shared files (schema.ts, nav.ts, i18n.ts, errors.ts). The generator stress tests found 6+ bugs caused by manual file wiring — this plan eliminates that entire class of errors.

## Why This Matters

Every generated feature needs to wire itself into 4 shared files. The current approach is fragile: Plop templates append to files using text concatenation, which breaks on edge cases (trailing commas, existing imports, multi-line expressions, comments). ts-morph manipulates the AST directly, producing correct code regardless of formatting.

## must_haves

- [ ] ts-morph installed as a project dependency
- [ ] Schema wirer: adds table definition import + spreads into defineSchema() call in convex/schema.ts
- [ ] Nav wirer: adds icon import + appends NavItem to navItems array in src/shared/nav.ts
- [ ] i18n wirer: adds namespace string to ns array in src/i18n.ts
- [ ] Errors wirer: adds error constant group to ERRORS object in src/shared/errors.ts
- [ ] All wiring operations are idempotent (running twice produces same result, no duplicates)
- [ ] All wiring operations include unwiring (remove feature from shared files)
- [ ] Tests verify wiring against actual project files (not mocked stubs)
- [ ] `npm test` passes at 100% coverage

Implementation details:

- Check if property already exists (idempotency) — skip if found

- Save file after modifications

- authTables spread (`...authTables`) already exists — add AFTER it
- Multiple existing features — append at end of object literal
- Feature already wired — skip silently (idempotent)
