# Plan 999.1-01: Feature YAML Schema — Zod-Validated feather.yaml Format

## Objective

Design and implement the complete `feather.yaml` schema format that serves as the single source of truth for feature generation. This is the foundation — every other plan depends on the YAML format being defined, validated, and documented. The schema covers all 8 dimensions of the "Beyond CRUD" framework plus the baseline Schema+CRUD.

## Why This Matters

The YAML format is Decision #8 in the architecture document: "Design feather.yaml schema format — the YAML format is the foundation everything else builds on." Every generator operation, every AST wiring step, every cross-entity UI declaration reads from this YAML. Getting the schema right means the entire framework has a stable contract.

## must_haves

- [ ] Zod v4 schema for the complete feather.yaml format covering all 8 dimensions
- [ ] Root `feather.yaml` project config schema (branding, features list, bundles, settings)
- [ ] Per-feature YAML schema with all dimension sections as opt-in
- [ ] Validation produces human-readable errors with YAML line number context
- [ ] Minimal YAML (just `name` + `fields`) produces a valid config after defaults merge
- [ ] `defaults.yaml` upgraded to cover all new dimension defaults
- [ ] Existing `tasks.gen.yaml` migrated to new format as `tasks/feather.yaml`
- [ ] Tests cover: valid minimal input, valid full input, invalid inputs for each dimension, defaults merging, relationship validation, status flow transition validation
- [ ] `npm test` passes at 100% coverage

<action>
Create `templates/schema/field.schema.ts` with Zod v4 schemas for feature fields:

Field types to support:
- `string` — text input, max length, optional pattern regex
- `text` — multiline textarea, max length
- `number` — numeric input, optional min/max, optional step
- `boolean` — toggle/checkbox, default value
- `enum` — predefined values array, optional transitions map, filterable flag
- `date` — date picker (stored as ISO string or number)
- `reference` — foreign key to another entity (v.id("tableName"))
- `email` — string with email validation
- `url` — string with URL validation
- `currency` — number with currency display config (symbol, precision)
- `percentage` — number displayed as percentage

Field properties common to all types:
- `required` (boolean, default false)
- `max` (number, optional — max length for string/text, max value for number)
- `min` (number, optional — min length for string/text, min value for number)
- `default` (matching type, optional)
- `showOn` ("all" | "form" | "detail" | "list", default "all")
- `hideOn` (array of view names, default [])
- `filterable` (boolean, default false)
- `sortable` (boolean, default true)
- `searchable` (boolean, default false)
- `label` (string, optional — override auto-generated label from field name)
- `description` (string, optional — help text for form fields)

For enum fields, additional properties:
- `values` (string array, required)
- `transitions` (record of state -> allowed next states, optional)
- `transitionGuards` (record of transition -> condition expression, optional)

Export: `fieldSchema`, `fieldTypeSchema`, `fieldsMapSchema`
</action>

```typescript
export const featureYamlSchema = z.object({
  name: z.string(),
  label: z.string(),
  labelPlural: z.string().optional(),  // auto-derived from label + "s" if not set

  // Baseline
  fields: fieldsMapSchema,
  timestamps: z.union([z.boolean(), z.enum(["both", "created", "updated", "none"])]).default("both"),
  identity: identitySchema.default({ type: "auto-increment" }),

  // 8 Dimensions (all optional — progressive disclosure)
  access: accessSchema.optional(),
  statusFlow: statusFlowSchema.optional(),
  hooks: hooksSchema.optional(),
  derivedData: z.record(z.string(), derivedDataItemSchema).optional(),
  views: viewsSchema.optional(),
  schedules: z.record(z.string(), scheduleSchema).optional(),
  integrations: z.record(z.string(), integrationSchema).optional(),

  // Relationships
  relationships: z.record(z.string(), relationshipSchema).optional(),
  children: z.record(z.string(), childEntitySchema).optional(),

  // Detail View Composition (D-04)
  detailView: detailViewSchema.optional(),

  // Operations
  operations: z.object({
    create: z.boolean().default(true),
    read: z.boolean().default(true),
    update: z.boolean().default(true),
    delete: z.boolean().default(true),
  }).default({}),

  // Behaviors (legacy compat + extensions)
  behaviors: z.object({
    assignable: z.boolean().default(false),
    orderable: z.boolean().default(false),
    softDelete: z.boolean().default(false),
    auditTrail: z.boolean().default(false),
  }).optional(),

  // Override pointers (D-08)
  overrides: z.record(z.string(), z.string()).optional(),

  // Search
  search: z.boolean().default(false),
  searchFields: z.array(z.string()).optional(),

  // Indexes
  indexes: z.array(z.object({
    name: z.string(),
    fields: z.array(z.string()),
  })).optional(),

  // i18n
  i18n: z.object({
    languages: z.array(z.string()).default(["en", "es"]),
  }).optional(),
});
```
