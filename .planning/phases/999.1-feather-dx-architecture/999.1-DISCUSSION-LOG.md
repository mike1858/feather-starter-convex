## Cross-Entity: Tight Parent-Child Location

| Option | Description | Selected |
|--------|-------------|----------|
| Inline in parent YAML | PO line items defined inside purchase-orders.yaml under children: | ✓ |
| Separate YAML with binding | Own YAML with parent: declaration, auto-removed together | |

**User's choice:** Inline in parent YAML

## Escape Hatch for Heavy Customization

| Option | Description | Selected |
|--------|-------------|----------|
| Copy to custom/ and edit | feather eject command | |
| Shadow file convention | Same filename in custom/ auto-replaces | |
| YAML override pointer | overrides: { form: custom/tasks/CustomForm.tsx } | ✓ |

**User's choice:** Undecided — asked Claude to pick best option and document alternatives.
**Notes:** User concern: "If we allow them to copy and then we auto generate things, whenever the YAML file changes, we might have conflicts." Claude chose YAML override pointer for explicitness and generator awareness.
