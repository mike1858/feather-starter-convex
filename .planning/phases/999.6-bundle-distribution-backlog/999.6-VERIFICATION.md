**Registry fetchFeatureFiles partial note:** fetchFeatureFiles (registry.ts lines 127-165) fetches and caches manifest.json but creates empty subdirectories rather than fetching actual feature template files (lines 156-163). The `subdirUrl` is constructed but never fetched. This means after a remote registry sync, `resolve()` will find the manifest and return type:"feature", but `installSingleFeature()` will find no template files to copy and produce an empty install. This is a documented limitation in the SUMMARY ("Known Stubs" section) and RESEARCH.md (Open Question 2 / files_flat field). It does not break the local-first story (the primary use case) but the remote-fallback install goal is partially hollow. Classified as WARNING rather than blocker — local bundle install works fully; remote fetch caches metadata only.
