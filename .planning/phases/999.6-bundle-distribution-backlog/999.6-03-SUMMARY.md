## Decisions Made
- addAction changed from synchronous to async to accommodate the await on fetchFromRegistry
- Registry fetch follows the same bundle-first → feature-second order as local resolve() for consistency
- Fetched manifests are cached to templates/ so the existing resolve() function finds them without changes
- syncRegistry uses the project's feather.yaml bundles/features lists to determine what to sync (no separate registry index needed for v1)
- Network errors are caught and wrapped in descriptive messages mentioning the URL and suggesting network checks
