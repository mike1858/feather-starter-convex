<task type="auto">
  <name>Task 1: Playwright setup + clearAll mutation + auth E2E tests</name>
  <files>playwright.config.ts, e2e/fixtures.ts, e2e/auth.spec.ts, convex/testing/clearAll.ts, package.json, vitest.config.ts</files>
  <action>
    1. Install Playwright: `npm install -D @playwright/test && npx playwright install chromium`

    2. Create `playwright.config.ts` at project root:
       - baseURL: `http://localhost:5173` (Vite dev server)
       - testDir: `./e2e`
       - Use Chromium only (faster local runs)
       - webServer config to start Vite dev server if not running:
         ```typescript
         webServer: {
           command: "npm run dev",
           port: 5173,
           reuseExistingServer: true,
         },
         ```
       - Set reasonable timeout (30s per test)
       - Configure retries: 0 (we want deterministic results)

    3. Create `convex/testing/clearAll.ts`:
       - Export a mutation that deletes all rows from ALL tables: users, plans, subscriptions, devEmails, and auth tables (authAccounts, authSessions, authVerificationCodes, authVerifiers, authRateLimits, authRefreshTokens)
       - Use `ctx.db.query(tableName).collect()` + `ctx.db.delete(doc._id)` for each table
       - This mutation is imported by the E2E fixture for auto-cleanup

    4. Create `e2e/fixtures.ts`:
       ```typescript
       import { createConvexTest } from "feather-testing-convex/playwright";
       import { api } from "../convex/_generated/api";

       export const test = createConvexTest({
         convexUrl: process.env.E2E_CONVEX_URL ?? process.env.VITE_CONVEX_URL!,
         clearAll: api.testing.clearAll,
       });
       export { expect } from "@playwright/test";
       ```
       - Use `E2E_CONVEX_URL` env var (separate deployment per locked decision)
       - Fall back to `VITE_CONVEX_URL` for development convenience

    5. Create `e2e/auth.spec.ts` with these tests:
       - **Sign up with email/password:** Visit /login, fill email + password, click "Sign up", assert redirect to /onboarding/username, complete onboarding, assert redirect to /dashboard
       - **Sign out:** After sign up + onboarding, click sign out, assert back at /login
       - **Sign in with email/password:** After sign up (in a prior test or using a reusable helper), visit /login, fill email + password, click "Sign in", assert redirect to /dashboard
       - **Password reset flow:** Sign up first, sign out, click "Forgot password?", enter email, submit. Then query the devEmails table (via a test-only query or by visiting the dev mailbox route and extracting the code), enter code + new password, assert success. Sign in with new password.

       For the password reset test, to get the reset code:
       - Option A: Visit /dev/mailbox route and extract the code from the email display
       - Option B: Use the Convex client from fixtures to query devEmails directly
       - Prefer Option A as it tests the dev mailbox integration too

       Create a reusable `signUp(session, email, password)` helper function that other spec files can import. This helper does: visit /login -> fill email + password -> click "Sign up" -> fill username -> click continue -> assert /dashboard.

    6. Update `vitest.config.ts`:
       - Add `convex/testing/**` to coverage excludes (test infrastructure, not production code)
       - The `e2e/` directory is not in the vitest include paths so it won't affect coverage

    7. Add npm script to `package.json`:
       ```json
       "test:e2e": "npx playwright test"
       ```
