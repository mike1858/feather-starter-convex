# Plan 999.3-01: Conversation State Checkpoint System

## Objective

Implement the JSON checkpoint system that persists conversation state between sessions. This is the foundation for the LLM architect skill — without reliable state persistence, the multi-phase conversation cannot survive session resets. The checkpoint captures entities discovered, YAML generated, behaviors configured, and sample data validated across all 4 conversation phases.

## Why This Matters

The conversation flows across 4 phases (Entity Discovery -> Schema+CRUD -> Behavior Overlay -> Validation) and may span multiple Claude Code sessions. D-07 requires JSON checkpoint files in `.feather/conversations/{id}.json`. D-08 requires non-linear navigation (revisiting earlier phases). D-09 requires cascade detection when revisiting. All of this depends on a well-defined, versioned state format.

## must_haves

- [ ] TypeScript types for the complete conversation state (all 4 phases)
- [ ] JSON checkpoint read/write with atomic file operations
- [ ] Conversation listing (find all conversations in `.feather/conversations/`)
- [ ] Resume logic: read checkpoint, determine current phase, summarize captured state
- [ ] Phase revisit support: set currentPhase backward, optionally clear downstream data
- [ ] Schema version field for future migration support
- [ ] Unit tests covering: create, save, load, resume, phase navigation, cascade clearing
- [ ] `npm test` passes at 100% coverage

1. Schema version constant: `CONVERSATION_SCHEMA_VERSION = 1`

2. Entity draft type (Phase 0 output):
```typescript
interface EntityDraft {
  name: string;           // kebab-case entity name (e.g., "work-logs")
  label: string;          // human-readable singular (e.g., "Work Log")
  labelPlural: string;    // human-readable plural (e.g., "Work Logs")
  description: string;    // what this entity represents
  fields: Record<string, { type: string; required?: boolean; max?: number; default?: unknown; values?: string[] }>;
  relationships: { type: "belongs_to" | "has_many"; target: string; required?: boolean; column?: string }[];
}
```

3. Behavior overlay type (Phase 2 output):
```typescript
interface BehaviorOverlay {
  access?: { scope: string; permissions: Record<string, string> };
  statusFlow?: { field: string; transitions: Record<string, string[]> };
  hooks?: Record<string, string>;
  derivedData?: Record<string, { type: string; source?: string; field?: string }>;
  views?: { defaultView: string; enabledViews: string[]; filteredViews?: unknown[] };
  schedules?: Record<string, unknown>;
  integrations?: Record<string, unknown>;
  identity?: { type: string; format?: string };
  behaviors?: { assignable?: boolean; orderable?: boolean; softDelete?: boolean; auditTrail?: boolean };
}
```

4. Main conversation state type:
```typescript
interface ConversationState {
  schemaVersion: number;
  id: string;                         // human-readable: "{domain}-{YYYY-MM-DD}"
  domain: string;                     // project/domain name
  createdAt: string;                  // ISO timestamp
  updatedAt: string;                  // ISO timestamp
  currentPhase: 0 | 1 | 2 | 3;       // which phase we're in

  // Phase 0: Entity Discovery
  entities: EntityDraft[];

  // Phase 1: Schema + CRUD
  yamlFiles: Record<string, string>;  // entityName -> generated YAML content
  validationResults: { entity: string; valid: boolean; errors: string[] }[];

  // Phase 2: Behavior Overlay
  behaviors: Record<string, BehaviorOverlay>;

  // Phase 3: Validation
  sampleData: Record<string, Record<string, unknown>[]>;
  dryRunResult: { success: boolean; errors: string[]; filesGenerated: string[] } | null;

  // Metadata
  phaseHistory: { phase: number; completedAt: string; entityCount: number }[];
}
```

5. Zod validation schema for the conversation state (for safe deserialization).

6. Factory function: `createConversation(domain: string): ConversationState` — creates a new state with Phase 0, empty arrays, generated ID `"{domain}-{YYYY-MM-DD}"`.

7. Phase transition helpers:
   - `advancePhase(state: ConversationState): ConversationState` — increment currentPhase, record in phaseHistory
   - `revisitPhase(state: ConversationState, targetPhase: 0 | 1 | 2 | 3, clearDownstream: boolean): ConversationState` — set currentPhase backward, optionally clear phases after target
   - `getPhaseLabel(phase: number): string` — returns "Entity Discovery" | "Schema + CRUD" | "Behavior Overlay" | "Validation"
   - `summarizeState(state: ConversationState): string` — human-readable summary of what's been captured so far

Export all types and functions.

<action>
Create `lib/conversation/conversation-io.ts` with:

1. Constants:
   - `CONVERSATIONS_DIR = ".feather/conversations"` (relative to project root)

2. File I/O functions:
   - `saveConversation(projectRoot: string, state: ConversationState): void`
     - Ensures `.feather/conversations/` directory exists (recursive mkdir)
     - Writes to `{CONVERSATIONS_DIR}/{state.id}.json`
     - Uses atomic write: write to `.tmp` file first, then rename (prevents corruption on crash)
     - Updates `state.updatedAt` before writing
     - Validates state with Zod schema before writing (catch bugs early)

   - `loadConversation(projectRoot: string, id: string): ConversationState | null`
     - Reads `{CONVERSATIONS_DIR}/{id}.json`
     - Returns null if file doesn't exist
     - Validates with Zod schema on load (detects corrupt/outdated files)
     - Checks `schemaVersion` — if older than current, log warning (future: migrate)

   - `listConversations(projectRoot: string): { id: string; domain: string; currentPhase: number; updatedAt: string }[]`
     - Reads all `.json` files in `{CONVERSATIONS_DIR}/`
     - Returns summary array sorted by `updatedAt` descending (most recent first)
     - Gracefully skips malformed files (log warning, don't crash)

   - `deleteConversation(projectRoot: string, id: string): boolean`
     - Deletes the JSON file, returns true if deleted, false if not found

3. `.gitignore` entry: Add `.feather/` to project `.gitignore` if not already present (conversation state is developer-local, not committed).

Export all functions.
</action>
