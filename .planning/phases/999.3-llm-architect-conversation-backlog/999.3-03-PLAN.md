# Plan 999.3-03: Web Preview Dashboard

## Objective

Build the web preview dashboard that opens in the developer's browser, showing a visual representation of the domain model as it forms through the conversation. The dashboard displays an ER diagram (entity relationships), syntax-highlighted YAML for each entity, sample data tables, and validation status. It updates between conversation phases with on-demand refresh support (D-01, D-02, D-03).

## Why This Matters

The dashboard is the "show, don't tell" component (D-02). Developers see their domain model forming visually, which is far more effective than reading YAML in the terminal. The ER diagram validates relationships at a glance. Sample data tables validate that the schema captures the right information. This is the Tableau Standard in action — the developer sees immediate, visual feedback.

## must_haves

- [ ] Mermaid.js ER diagram generated from conversation entities and relationships
- [ ] Syntax-highlighted YAML display for each entity's feather.yaml
- [ ] Sample data tables rendered as HTML tables
- [ ] Validation status badges (pass/fail) per entity
- [ ] Auto-refresh via file polling (no server dependency)
- [ ] Single HTML file with no build step (CDN dependencies: mermaid.js, prism.js)
- [ ] Preview generator callable from skill via `npx tsx lib/conversation/preview-generator.ts {id}`
- [ ] Dashboard renders correctly in Chrome, Firefox, Safari
- [ ] Unit tests for mermaid diagram generation and HTML generation
- [ ] `npm test` passes at 100% coverage

   ```
           string name
           enum status
       }
       TASKS {

           text description
           enum status
           boolean priority
       }
       SUBTASKS {
           string title
           boolean done
       }
   ```
