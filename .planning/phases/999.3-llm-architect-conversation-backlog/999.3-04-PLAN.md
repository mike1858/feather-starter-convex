# Plan 999.3-04: Integration Testing — CalmDo Domain End-to-End Validation

## Objective

Validate the complete LLM architect conversation pipeline by running it against the CalmDo domain (5 entities: tasks, projects, subtasks, work-logs, activity-logs). This plan verifies that entity discovery, YAML generation, behavior overlay, sample data, and validation all work together to produce correct, generator-compatible feather.yaml files. The CalmDo domain is the most complex real-world test case available in this project.

## Why This Matters

Individual units (state management, YAML generation, preview) have been tested in isolation. This plan tests the complete pipeline: entity drafts -> YAML generation -> validation -> behavior overlay -> re-validation -> sample data -> dry-run generation. It also validates that the generated YAML matches or exceeds the quality of the existing hand-written feather.yaml files.

## must_haves

- [ ] CalmDo domain fixture with all 5 entities, their fields, and relationships
- [ ] Integration test that exercises the full conversation pipeline (Phase 0 -> 1 -> 2 -> 3)
- [ ] Generated YAML for each CalmDo entity validates against featureYamlSchema
- [ ] Generated YAML includes correct relationships between entities
- [ ] Behavior overlay adds access control, hooks, derived data to tasks entity
- [ ] Sample data generated for all entities with correct field types
- [ ] Dry-run generation succeeds (or documents any expected gaps)
- [ ] Preview HTML renders with ER diagram showing all 5 entities connected
- [ ] Comparison of generated YAML vs existing feather.yaml files (gap report)
- [ ] Skill file updated with any fixes found during integration testing
- [ ] `npm test` passes at 100% coverage
