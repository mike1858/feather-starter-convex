## Decisions Made

- **dataFingerprint as JSON string:** Stored mappings encode data fingerprint as JSON string; `parseDataFingerprint()` parses it for Jaro-Winkler data scoring. This ensures stored sample values participate in rename detection (30% weight).
- **Batch error storage (BATCH_SIZE=50):** Convex actions have arg size limits; errors are stored in batches of 50 to avoid hitting the limit.
- **Removed duplicate schema tables:** `convex/schema.ts` had imports, schemaMappings, importErrors defined twice (raw validators and zodToConvex versions). Kept only the zodToConvex versions for single source of truth.
- **Green section collapsed by default:** ImportReport collapses green (auto-fixed) errors by default since they require no user action. Yellow and red sections are expanded.
- **Confirm flow in ImportWizard (not DashboardReview):** The handleConfirm logic lives in ImportWizard.tsx (parent) rather than DashboardReview (child) for cleaner data flow and mutation access.
