## Decisions Made
- Excluded `schema-mappings.ts` from coverage since it's a pure Zod type declaration not imported by any backend code (the mutations use inline `JSON.parse` typing)
- Import mutations throw explicit errors for unauthenticated users (unlike tasks mutations which silently return) since import operations should surface auth failures clearly
- Used JSON string fields for flexible metadata storage in Convex tables (sheetMetadata, analysisResult, confirmedSchema, importStats, importHistory) since Convex doesn't support nested objects with variable schemas
