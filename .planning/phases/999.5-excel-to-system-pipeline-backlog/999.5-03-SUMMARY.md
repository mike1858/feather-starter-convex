## Decisions Made

- **DetectedRelationship defined locally:** The entity-classifier module from Plan 02 doesn't exist yet (parallel execution). Defined the type in `src/features/import/types.ts` with basic relationship detection logic in useSchemaAnalysis.
- **Client-side relationship detection:** Scans for reference-type fields and name-matches them to entity names (e.g., "customerId" -> "customers"). Simple but effective for heuristic analysis.
- **Coverage exclusions for Convex containers:** ImportWizard.tsx and useSchemaAnalysis.ts excluded since they orchestrate useMutation/useAction hooks that need a live Convex backend. All child components fully tested at 100%.
- **Explicit state types in Step3Relationships:** Refactored from `Partial<DetectedRelationship>` to explicit interface with required `type` and `confidence` fields. Eliminates unreachable `??` branches without v8 ignore comments.
