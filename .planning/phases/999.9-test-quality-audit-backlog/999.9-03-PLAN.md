# Plan 999.9-03: Test Philosophy Re-Audit — Fix Violations Across All Test Files

## Objective

Fix all must-fix and should-fix violations identified in Plan 01's test philosophy audit (Task 999.9-01-03). This covers the 47 non-generator test files — backend (22 files) and frontend/shared/UI (25 files). After this plan, every test file follows the 12-point testing philosophy checklist.

## Why This Matters

Phase 7 rewrote test suites for MECE compliance, but D-08 mandates a full re-audit because: (a) Phase 7 was done under time pressure, (b) subsequent phases (4, 5, 6) added new tests that weren't held to the same standard, (c) generated example app tests followed template patterns, not hand-reviewed quality. This plan closes the gap.

## must_haves
- [ ] All must-fix violations from findings report resolved
- [ ] All should-fix violations from findings report resolved
- [ ] No `toBeDefined()` as sole assertion in any test file
- [ ] No `toMatchSnapshot()` in any test file
- [ ] No `expect(true).toBe(true)` placeholders in any test file
- [ ] All backend test files have MECE test matrix comments
- [ ] All test names follow `"verb what-user-sees [condition]"` convention
- [ ] `npm test` passes with 100% coverage
- [ ] No test depends on another test's execution order

<action>
Fix all violations identified in the findings report for the 9 core feature backend test files (tasks, projects, subtasks, workLogs, activityLogs). Common fixes based on typical Phase 7 gaps:

**Anti-pattern fixes:**
1. Replace any `toBeDefined()` sole assertions with specific value checks: `expect(task.title).toBe("Test Task")` not `expect(task).toBeDefined()`
2. Replace any `toMatchSnapshot()` with explicit assertions
3. Remove any `expect(true).toBe(true)` placeholders
4. Ensure no test verifies mock call counts (`expect(mockFn).toHaveBeenCalledTimes(1)`) — assert behavior instead

**Naming fixes:**
5. Rename any tests starting with "should" to verb-noun format: `"should create a task"` → `"creates task with default status and visibility"`
6. Ensure test names describe what the user sees, not internal implementation

**MECE fixes:**
7. Add `// Test Matrix:` comment at top of any file missing it
8. Check for overlapping tests (two tests that verify the same state) — merge into one with multiple assertions
9. Check for missing states (MECE gap) — add tests for uncovered states

**Independence fixes:**
10. Ensure no test relies on state from a previous test (shared mutable state, test ordering)
11. Each test should set up its own data via mutations, not rely on other tests' side effects

For each file, consult the specific findings in `999.9-FINDINGS.md` — the findings report lists exact line numbers and issues.

Run `npm test` after fixing each file (or batch of related files).
</action>
<acceptance_criteria>
- All 9 core backend test files rated "pass" against the 12-point checklist
- Zero `toBeDefined()` as sole assertion across these files
- All test names follow verb-noun convention (no "should" prefix)
- All files have `// Test Matrix:` comment
- `npm test` passes with 100% coverage
