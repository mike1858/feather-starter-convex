       - Renders a clickable badge showing current status with color coding (todo=gray, in_progress=blue, done=green)
       - On click, calls `updateStatus` mutation with next status (todo->in_progress, in_progress->done)
       - When status is "done", click does nothing (or shows checkmark)
       - Use lucide icons: Circle for todo, ArrowRight or Loader for in_progress, CheckCircle for done

    3. Create `src/features/tasks/components/TaskForm.tsx`:
       - Inline form with title input (required, max 200 chars) and submit button
       - Uses @tanstack/react-form with Zod validation from createTaskInput schema
       - On submit, calls `create` mutation and clears the form
       - Minimal: just title field for quick-add. Description/priority can be edited after creation.
       - Placeholder text: "Add a task..." with keyboard shortcut hint

    4. Create `src/features/tasks/components/TaskItem.tsx`:
       - Renders a single task row: drag handle | status badge | title | priority indicator (flag icon if high) | assignee avatar/name | delete button
       - Priority toggle: click flag to toggle priority via `update` mutation
       - Assignee: show avatar or initial, click to open a simple dropdown of team members (from `listUsers` query), select to call `assign` mutation
       - Delete: use `useDoubleCheck` hook pattern from SettingsPage. First click shows "Are you sure?", second click calls `remove` mutation.
       - Edit title: click title to make it editable inline (contentEditable or input swap), blur/enter saves via `update` mutation

    5. Create `src/features/tasks/components/TaskList.tsx`:
       - Wraps tasks in dnd-kit `DndContext` + `SortableContext` with `verticalListSortingStrategy`
       - Each TaskItem wrapped in a SortableTaskItem using `useSortable` hook
       - On drag end: calculate new position as midpoint between neighbors (or +/- 1000 for edges), call `reorder` mutation
       - If position gap < 1 after reorder, trigger normalization (reassign positions 1000, 2000, 3000...)
       - Show empty state message when no tasks
       - Sort tasks by position ascending before rendering

    6. Create `src/features/tasks/components/TasksPage.tsx` (My Tasks):
       - Queries `api.tasks.queries.myTasks` via `useQuery(convexQuery(...))`
       - Shows TaskForm at top for quick task creation
       - Shows TaskList below with user's tasks
       - Page title: "My Tasks"

    7. Create `src/features/tasks/components/TeamPoolPage.tsx` (Team Pool):
       - Queries `api.tasks.queries.teamPool` via `useQuery(convexQuery(...))`
       - Shows TaskList with unassigned shared tasks
       - NO TaskForm (you create tasks from My Tasks, they appear here when unassigned+shared)
       - Include a "Grab" button on each task item that assigns it to current user (calls `assign` with current userId)
       - Page title: "Team Pool"
