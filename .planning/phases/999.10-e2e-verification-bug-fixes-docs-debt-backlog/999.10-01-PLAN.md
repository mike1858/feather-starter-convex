# Plan 999.10-01: E2E Test Verification and CalmDo Feature E2E Tests

## Objective

Verify existing E2E tests pass against the live dev app, then write new E2E tests covering all CalmDo user flows: task CRUD with status transitions, project management, subtasks, work logs, and the activity timeline. This is the final quality gate before team use.

## Why This Matters

E2E tests were written in Phase 02-04 but never run against a live deployment (deferred as "Task 3 of 02-04"). Unit tests (489 passing) prove logic correctness but cannot verify real browser rendering, Convex reactivity, form submissions, or navigation flows. This plan closes that gap.

## must_haves
- [ ] All 3 existing E2E spec files (auth, onboarding, settings) pass against local dev server
- [ ] Task CRUD flow tested: create task, edit title, change status (todo -> in_progress -> done), delete with confirmation
- [ ] Project flow tested: create project, view project detail, assign task to project
- [ ] Task detail panel tested: add subtask, mark subtask done, add work log entry, view activity timeline
- [ ] All E2E tests pass: `npm run test:e2e` exits 0

<task id="999.10-01-02">
<title>Write E2E tests for task CRUD and status workflow</title>
<type>e2e-test</type>
<read_first>
- e2e/helpers.ts (existing signUp helper — extend with task creation helpers)
- e2e/fixtures.ts (test fixtures)
- src/features/tasks/components/TaskPage.tsx (task list UI — element selectors, button labels)
- src/features/tasks/components/TaskForm.tsx (task creation form — field names, submit button)
- src/features/tasks/components/TaskItem.tsx (task list item — status badge, edit/delete buttons)
- convex/tasks/mutations.ts (available mutations: create, update, remove, updateStatus, assign, unassign)
- src/shared/schemas/tasks.ts (Zod schema — field names and constraints)
- public/locales/en/tasks.json (i18n keys — button labels, titles)
</read_first>
<action>
Create `e2e/tasks.spec.ts` with the following tests:

1. **Create a task** — Navigate to My Tasks, click "New Task" (or equivalent create button per i18n), fill in title "Test Task E2E", submit, verify task appears in the list.

2. **Edit a task** — Create a task, click on it to open detail/edit, change title to "Updated Task E2E", save, verify updated title appears in list.

3. **Status transitions** — Create a task (starts as `todo`), change status to `in_progress`, verify status badge updates, change to `done`, verify status badge shows done.

4. **Delete a task with confirmation** — Create a task, click delete, confirm in the confirmation dialog, verify task no longer appears in list.

5. **Task visibility** — Create a quick task (should be private by default), verify it appears in "My Tasks" but not in "Team Pool".

Use the existing `signUp` helper to authenticate before each test. Create a `createTask` helper in `e2e/helpers.ts` for reuse:

```typescript
export function createTask(session: Session, title: string): Session {
  return session
    .click("New Task")  // or whatever the actual button label is
    .fillIn("Title", title)
    .clickButton("Create")  // or "Save"
    .assertText(title);
}
```

Verify actual button labels and element structure by reading the component files first — do not assume labels.
</action>
<acceptance_criteria>
- `e2e/tasks.spec.ts` exists with at least 4 tests in a `test.describe("Task flows")` block
- Each test authenticates via `signUp` helper
- Task creation test asserts the new task title appears on the page
- Status transition test verifies the status badge text changes
- Delete test verifies the task is removed from the list
- `npm run test:e2e` passes with all task tests green
</acceptance_criteria>
</task>
