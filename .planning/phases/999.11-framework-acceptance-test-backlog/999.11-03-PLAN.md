## Context

The journey test proves the entire app hangs together: signup works, navigation between features works, each feature's inline form works, Convex reactivity delivers items, and no routing errors occur. It is a single test that walks through the complete user experience.

**Helpers available (after Plan 2):**
- `signUp(session, email, password, username)` — full signup + onboarding, ends at `/dashboard`
- `createTask(session, title)` — fill "Add a task..." + click "Add" + assertText
- `createProject(session, name)` — visit projects, fill "New project name..." + click "Create" + assertText
- `createTodo(session, title)` — fill "Add todo..." + click "Add" + assertText
- `createTicket(session, title)` — fill "Add ticket..." + click "Add" + assertText
- `createContact(session, name)` — fill "Add contact..." + click "Add" + assertText

**Routes to visit and create items:**
| Route | Action | Helper |
|-------|--------|--------|
| `/dashboard` | Verify landing page loads | `assertText("Dashboard")` |
| `/dashboard/tasks` | Create a task | `createTask` |
| `/dashboard/projects` | Create a project | `createProject` |
| `/dashboard/todos` | Create a todo | `createTodo` |
| `/dashboard/tickets` | Create a ticket | `createTicket` |
| `/dashboard/contacts` | Create a contact | `createContact` |
| `/dashboard/settings` | Verify page loads | `assertText("Your Username")` |

**Routes to SKIP:**
- Subtasks, Work Logs, Activity Logs — sub-entities accessed via task detail panel, no dedicated CRUD pages
- Import, Imports — may not have dedicated route files
- Team Pool — empty state, no create action, not worth testing in journey

## Tasks

- [ ] **Task 3.1:** Create the journey E2E spec

  Create `e2e/journey.spec.ts` with a single comprehensive test:

  ```typescript
  import { test, expect } from "./fixtures";
  import {
    signUp,
    createTask,
    createProject,
    createTodo,
    createTicket,
    createContact,
    uniqueEmail,
  } from "./helpers";

  test.describe("Full navigation journey", () => {
    test("signup through every feature", async ({ session }) => {
      const email = uniqueEmail();
      await signUp(session, email, "password123", "journeyuser");

      // 1. Dashboard loads after signup
      await session.assertPath("/dashboard");

      // 2. Tasks — create an item
      await session.visit("/dashboard/tasks");
      await session.assertText("My Tasks");
      await createTask(session, "Journey Task E2E");

      // 3. Projects — create an item
      // (createProject visits /dashboard/projects internally)
      await createProject(session, "Journey Project E2E");

      // 4. Todos — create an item
      await session.visit("/dashboard/todos");
      await session.assertText("Todos");
      await createTodo(session, "Journey Todo E2E");

      // 5. Tickets — create an item
      await session.visit("/dashboard/tickets");
      await session.assertText("Tickets");
      await createTicket(session, "Journey Ticket E2E");

      // 6. Contacts — create an item
      await session.visit("/dashboard/contacts");
      await session.assertText("Contacts");
      await createContact(session, "Journey Contact E2E");

      // 7. Settings — verify page loads
      await session.visit("/dashboard/settings");
      await session.assertText("Your Username");

      // 8. Return to dashboard — verify it loads
      await session.visit("/dashboard");
      await session.assertPath("/dashboard");
    });
  });
  ```

  **Important notes:**
  - Single test, single user session — proves the whole app works in sequence for one user
  - `createProject` already calls `session.visit("/dashboard/projects")` internally, so no need to visit separately
  - Each `createX` helper calls `assertText(title)` which confirms the item was created and appears
  - No `.skip` or `.only` allowed
  - Use a unique username suffix to avoid collision with parallel test runs

  **Files:** `e2e/journey.spec.ts`
  **Verify:** `npx playwright test e2e/journey.spec.ts` passes
