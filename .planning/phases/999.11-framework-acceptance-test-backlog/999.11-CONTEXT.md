# Phase 999.11: Framework Acceptance Test — Full User Journey E2E

## Why This Phase Exists

All 21 v2.0 phases are code-complete with 1184 unit/integration tests passing. But no test verifies the **full user journey**: new project creation → example app installation → browser CRUD working. Unit tests with mocks can't catch:

- Convex modules not deploying (hyphenated dirs, stale codegen)
- Generated code that compiles but doesn't render interactively
- Wiring gaps (nav entries, schema tables, i18n namespaces) that only surface at runtime
- Import/build errors in scaffolded projects that don't run the full Vite pipeline

The user's requirement: **until a new user can install the starter, create example apps, and verify they work in the browser — the milestone isn't done.**

## Architecture Decisions

### Why NOT two parallel sub-agents generating projects
1. `create-feather` clones from GitHub, needs Convex credentials, runs `npm install` + `npx convex dev --once` — can't run safely in a sub-agent worktree
2. Each test project needs its own Convex deployment — can't parallelize without multiple instances
3. Dev server runs on localhost:5173 — only one project at a time

### Chosen approach: Two-layer verification

**Layer 1 (Plan 1): Offline pipeline test**
- Runs in CI, no network needed
- Extends existing `integration.test.ts`
- After strip→add: runs `tsc --noEmit` and `vite build` on the temp project
- Catches: missing imports, type errors, broken CSS/Tailwind, module resolution failures

**Layer 2 (Plans 2+3): Live Playwright E2E**
- Runs against the current project's dev server (already has all features installed)
- Plan 2: CRUD smoke tests per example app (todos, tickets, contacts)
- Plan 3: Full journey test (signup → onboard → visit every nav item → create item per feature)
- Catches: runtime rendering issues, Convex reactivity, form submission, navigation

### Why Playwright over agent-browser
- **Repeatable** — same test, same result, runs in CI
- **Fast** — 10x faster than agent-browser roundtrips
- **Permanent regression gate** — catches future breakage automatically
- agent-browser is great for one-off exploratory checks; Playwright is for automated gates
