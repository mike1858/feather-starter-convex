    **Part A: Create `src/features/tasks/tasks.gen.yaml`** (per D-37)

    This YAML retroactively documents the tasks data model. It should capture the existing implementation:

    ```yaml
    name: tasks
    label: Task
    labelPlural: Tasks

    fields:
      title:
        type: string
        required: true
        max: 200
      description:
        type: text
        max: 5000
      status:
        type: enum
        values: [todo, in_progress, done]
        default: todo
        filterable: true
        transitions:
          todo: [in_progress]
          in_progress: [done]
      visibility:
        type: enum
        values: [private, shared]
        default: private
        filterable: true
      priority:
        type: boolean
        default: false

    timestamps: false  # tasks uses _creationTime from Convex, not explicit timestamps

    behaviors:
      assignable: true
      orderable: true
      softDelete: false
      auditTrail: false
      immutable: false

    access:
      scope: custom  # tasks has complex visibility rules (private vs shared)
      permissions:
        create: authenticated
        read: custom  # my tasks vs team pool logic
        update: owner
        delete: owner

    views:
      default:
        defaultView: list
        enabledViews: [list]
      list:
        density: comfortable
      filteredViews:
        - name: my-tasks
          label: My Tasks
          filter: { assigneeId: $currentUser }
          navEntry: true
        - name: team-pool
          label: Team Pool
          filter: { visibility: shared, assigneeId: null }
          navEntry: true

    relationships:
      project:
        type: belongs_to
        target: projects
        required: false
        column: projectId

    indexes:
      - name: by_assignee
        fields: [assigneeId]
      - name: by_visibility
        fields: [visibility]
      - name: by_creator
        fields: [creatorId]
      - name: by_assignee_status
        fields: [assigneeId, status]
    ```

    NOTE: This YAML is documentation only — it does NOT replace the hand-written tasks code. Per D-37, the tasks feature has domain-specific customizations (visibility auto-flip, custom query logic) that go beyond what the generator produces. The YAML documents the data model so future developers understand the structure.
