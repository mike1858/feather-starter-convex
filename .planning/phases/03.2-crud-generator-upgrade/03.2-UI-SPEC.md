# Phase 03.2 — UI Design Contract

> Visual and interaction contract for generated frontend components. This phase produces Handlebars templates that output React components. The contract defines what those templates MUST produce so every generated feature is visually consistent with the hand-built tasks feature.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (manual shadcn pattern -- CSS variables + Radix primitives + CVA) |
| Preset | not applicable (`components.json` does not exist; tokens defined in `src/index.css`) |
| Component library | Radix UI primitives (wrapped in `src/ui/`) |
| Icon library | lucide-react |
| Font | system-ui, sans-serif (set in `src/index.css` body rule) |

**Note:** This phase does not add new UI primitives. Generated components compose existing `src/ui/` components (Button, Input, Select, Switch) and Tailwind utility classes. The design contract specifies what the **generated output** must look like, not new design decisions.

---

## Spacing Scale

Declared values (must be multiples of 4). Derived from existing tasks feature patterns:

| Token | Value | Usage in Generated Code |
|-------|-------|------------------------|
| xs | 4px (`gap-1`, `p-1`) | Icon-to-label gaps inside badges, icon button padding |
| sm | 8px (`gap-2`, `p-2`) | Item-to-item spacing in lists, inline form element gaps, item internal padding (py-2) |
| md | 16px (`gap-4`, `p-4`) | Card internal padding, form field vertical spacing |
| lg | 24px (`gap-6`, `p-6`) | Section-to-section spacing on page (heading-to-form, form-to-list), page horizontal padding |
| xl | 32px (`gap-8`) | Not used in current patterns -- reserved for future page-level layout |
| 2xl | 48px (`py-12`) | Header vertical padding (existing Header component) |
| 3xl | 64px | Not used in current patterns -- reserved |

Exceptions:
- Touch targets: status badge buttons use `px-2 py-1` (28px height minimum) -- acceptable for desktop-first app. Generated components follow the same compact sizing.
- Drag handles: 16x16px icon (`h-4 w-4`) with no extra padding -- matches existing TaskItem pattern.

## Typography

Derived from existing `src/index.css` body rule and tasks feature components. Two weights only: regular (400) for body text, semibold (600) for labels, headings, and structural emphasis.

| Role | Size | Weight | Line Height | Tailwind Class | Used In Generated Code |
|------|------|--------|-------------|---------------|----------------------|
| Body | 14px | 400 (regular) | 1.5 (default) | `text-sm` | Item titles, form inputs, descriptions, table cells |
| Label | 12px | 600 (semibold) | 1.5 | `text-xs font-semibold` | Status badges, enum badges, field labels, metadata, section labels, card titles, table column headers |
| Heading | 24px | 600 (semibold) | 1.2 | `text-2xl font-semibold` | Page title (e.g., "My Projects") |

**Rationale:** The existing TasksPage uses `text-2xl font-semibold` for the page heading and `text-sm` throughout for body content. The Header component uses `text-3xl font-medium` but that renders in the shared layout header, not inside generated page components. Generated pages use `text-2xl font-semibold` for their own `<h1>`. The previous 4-role / 3-weight table has been consolidated to 3 roles / 2 weights by merging Subheading into Label (both serve the same structural emphasis purpose at small sizes).

## Color

The project uses ShadCN's HSL CSS variable system. Generated components MUST use semantic token classes, not raw hex/HSL values, except for enum badge colors.

| Role | Token | Tailwind Class | Usage in Generated Code |
|------|-------|---------------|----------------------|
| Dominant (60%) | `--background` | `bg-background` | Page background, form background |
| Secondary (30%) | `--card` | `bg-card` | List item cards, detail page cards, table rows |
| Accent (10%) | `--primary` | `text-primary`, `bg-primary` | Page heading text, item title text, CTA button fill |
| Destructive | `--destructive` | `text-destructive` | Delete button icon and confirmation text only |
| Muted | `--muted` | `bg-muted`, `text-muted-foreground` | Disabled states, secondary text, placeholder text |
| Border | `--border` | `border-border` | Item card borders, input borders, section dividers |

**Accent reserved for:** Primary CTA button (`bg-primary text-primary-foreground`), page heading text (`text-primary`), item title text (`text-primary`), focus rings (`ring-ring`). Never used for decorative backgrounds.

**Opacity variants used in existing code (follow same pattern in generated code):**
- `text-primary/60` -- description text, secondary labels
- `text-primary/50` -- empty state text
- `text-primary/40` -- drag handle inactive, unassigned placeholder
- `text-primary/20` -- inactive toggle icon (e.g., low-priority flag)
- `hover:bg-primary/5` -- subtle hover background on interactive elements

### Enum Badge Color Palette

Generated enum badges use a round-robin palette. Colors are assigned in declaration order from the YAML `values` array. Each badge uses background + text pairs for light mode, with dark mode variants.

| Slot | Light BG | Light Text | Dark BG | Dark Text |
|------|----------|------------|---------|-----------|
| 0 | `bg-gray-100` | `text-gray-700` | `dark:bg-gray-800` | `dark:text-gray-300` |
| 1 | `bg-blue-100` | `text-blue-700` | `dark:bg-blue-900` | `dark:text-blue-300` |
| 2 | `bg-green-100` | `text-green-700` | `dark:bg-green-900` | `dark:text-green-300` |
| 3 | `bg-amber-100` | `text-amber-700` | `dark:bg-amber-900` | `dark:text-amber-300` |
| 4 | `bg-red-100` | `text-red-700` | `dark:bg-red-900` | `dark:text-red-300` |
| 5 | `bg-purple-100` | `text-purple-700` | `dark:bg-purple-900` | `dark:text-purple-300` |
| 6 | `bg-pink-100` | `text-pink-700` | `dark:bg-pink-900` | `dark:text-pink-300` |
| 7 | `bg-cyan-100` | `text-cyan-700` | `dark:bg-cyan-900` | `dark:text-cyan-300` |

**Override:** YAML `colors:` map on an enum field overrides auto-assignment per value. Example: `status: { colors: { active: 2, archived: 0 } }` assigns slot 2 (green) to "active" and slot 0 (gray) to "archived".

**Badge markup pattern:** `<span className="inline-flex items-center gap-1 rounded px-2 py-1 text-xs font-semibold {bgClass} {textClass} {darkClasses}">`

## Component Visual Contracts

These are the visual specifications each generated component type MUST follow. All patterns match the existing tasks feature.

### Page Component (`{PascalName}Page.tsx`)

```
Layout: flex flex-col gap-6, full width
  Heading section: flex flex-col gap-2
    h1: text-2xl font-semibold text-primary
    p: text-sm text-primary/60
  Form section (inline create)
  List/Card/Table section
```

### List View (`{PascalName}ListView.tsx`)

```
Container: flex flex-col gap-2
  Items: map over sorted data, render {PascalName}Item
  Empty state: centered text paragraph
```

### Card View (`{PascalName}CardView.tsx`)

```
Container: grid grid-cols-1 gap-4 sm:grid-cols-2 lg:grid-cols-3
  Cards: rounded-lg border border-border bg-card p-4 flex flex-col gap-2
    Title: text-sm font-semibold text-primary (truncate)
    Fields: text-xs text-primary/60
    Status badge: bottom-left
    Actions: bottom-right (edit, delete with aria-label="Delete {label}")
  Empty state: centered text paragraph
```

### Table View (`{PascalName}TableView.tsx`)

```
Container: w-full overflow-x-auto
  Table: w-full border-collapse
    thead: text-left text-xs font-semibold text-muted-foreground uppercase tracking-wider
      th: px-4 py-3 border-b border-border
    tbody:
      tr: hover:bg-muted/50 transition
        td: px-4 py-3 text-sm text-primary border-b border-border
  Pagination: flex items-center justify-between mt-4
    Info: text-sm text-muted-foreground ("Showing 1-20 of 45")
    Controls: flex items-center gap-2 (Previous / page numbers / Next buttons)
```

### Item Component (`{PascalName}Item.tsx`)

```
Container: flex items-center gap-2 rounded-md border border-border bg-card px-3 py-2
  Drag handle (if orderable): cursor-grab text-primary/40 hover:text-primary/60, GripVertical h-4 w-4
  Status badge (if enum with transitions)
  Title: flex-1 text-sm text-primary, click-to-edit inline
  Priority flag (if boolean priority field): toggle, orange-500 active / primary/20 inactive, Flag h-4 w-4
  Assignee selector (if assignable): Select w-32 h-8 text-xs
  Delete button: Button ghost sm, text-destructive, Trash2 h-4 w-4, aria-label="Delete {label}", useDoubleCheck confirmation
```

### Form Component (`{PascalName}Form.tsx`)

```
Layout: flex items-center gap-2 (inline create variant)
  Text input: Input component, flex-1 bg-transparent, placeholder "Add a {label}..."
  Submit button: Button sm, label "Add {Label}"

Full form variant (create/edit dialog or page):
  Layout: flex flex-col gap-4
    Each field: flex flex-col gap-1
      Label: text-sm font-semibold text-primary
      Input: mapped by field type:
        string -> Input component
        text -> textarea (rounded-md border border-input bg-background px-3 py-2 text-sm, min-h-[80px])
        boolean -> Switch component from src/ui/switch.tsx
        number -> Input type="number"
        enum -> Select component from src/ui/select.tsx
    Actions row: flex items-center gap-2 justify-end
      Cancel: Button variant="outline" size="sm"
      Submit: Button size="sm", label "Create {Label}" or "Save {Label}"
```

### Status Badge Component (`{PascalName}StatusBadge.tsx`)

```
Pattern: button with icon + label text
  Classes: flex items-center gap-1 rounded px-2 py-1 text-xs font-semibold transition hover:bg-primary/5 disabled:cursor-default disabled:opacity-70
  Icon: h-4 w-4, from lucide-react (auto-selected per status semantic or custom in YAML)
  Click behavior: advances to next status in transition chain (if transitions defined)
  Disabled: when no next transition available
  Title attribute: "Mark as {nextStatus}" or current label when terminal
```

### View Switcher

```
Container: flex items-center gap-1 rounded-md border border-border p-1 bg-muted
  Buttons (one per enabled view type):
    Inactive: px-3 py-1.5 text-xs font-semibold text-muted-foreground rounded hover:text-primary transition, aria-label="{viewType} view"
    Active: px-3 py-1.5 text-xs font-semibold text-primary bg-background rounded shadow-sm, aria-label="{viewType} view"
  Icons: h-4 w-4, from lucide-react:
    list -> List icon, aria-label="List view"
    card -> LayoutGrid icon, aria-label="Card view"
    table -> Table icon, aria-label="Table view"
    cover -> Image icon, aria-label="Cover view"
```

### Detail Page (`{PascalName}DetailPage.tsx`)

```
Layout: flex flex-col gap-6 max-w-2xl
  Header: flex items-center justify-between
    Title: text-2xl font-semibold text-primary
    Actions: flex items-center gap-2 (Edit button, Delete button with aria-label="Delete {label}")
  Fields section: flex flex-col gap-4
    Each field: flex flex-col gap-1
      Label: text-xs font-semibold text-muted-foreground uppercase tracking-wide
      Value: text-sm text-primary
  Children section (if children relationship):
    Section heading: text-sm font-semibold text-primary, border-b border-border pb-2 mb-2
    Inline grid (editable table for children)

```

## Copywriting Contract

All copy is generated into i18n JSON files (`public/locales/{en,es}/{name}.json`). The English defaults below define the pattern. The generator fills placeholders with `{Label}` (singular) and `{LabelPlural}` (plural) from YAML.

| Element | English Copy Pattern | Example (for "Project") |
|---------|---------------------|------------------------|
| Primary CTA (inline) | "Add {Label}" | "Add Project" |
| Primary CTA (full form) | "Create {Label}" | "Create Project" |
| Save CTA | "Save {Label}" | "Save Project" |
| Empty state (list) | "No {labelPlural} yet. Create one above!" | "No projects yet. Create one above!" |
| Empty state (filtered) | "No {labelPlural} match the current filter." | "No projects match the current filter." |
| Error: not found | "{Label} not found." | "Project not found." |
| Error: invalid transition | "Invalid status transition." | "Invalid status transition." |
| Error: auth required | "You must be signed in to perform this action." | "You must be signed in to perform this action." |
| Delete confirmation | "Are you sure?" (inline toggle via `useDoubleCheck`) | "Are you sure?" |
| Page heading | "{LabelPlural}" or view-specific label from YAML | "My Tasks", "Team Pool", "Projects" |
| Page description | "Manage your {labelPlural}." or view-specific description from YAML | "Manage your projects." |
| Nav entry label | "{LabelPlural}" (default) or custom from YAML `views.filteredViews[].label` | "Active Projects" |

**Destructive actions in this phase:**
- `remove` mutation -- Delete a record. Confirmation: inline `useDoubleCheck` toggle ("Are you sure?" replaces delete icon). No modal dialog. Matches existing TaskItem pattern.
- No bulk delete. No cascade delete confirmation in the generated UI (cascade happens silently in the backend mutation).

**Accessibility labels for icon-only interactive elements:**
- Delete button (Trash2 icon): `aria-label="Delete {label}"` -- e.g., `aria-label="Delete Project"`
- View switcher buttons (icon-only): `aria-label="{viewType} view"` -- e.g., `aria-label="List view"`, `aria-label="Card view"`, `aria-label="Table view"`, `aria-label="Cover view"`

## Interaction Contracts

### Click-to-Edit Title
- Single click on title text enters edit mode (inline `<input>`)
- Enter saves, Escape cancels, blur saves
- Empty input on save reverts to original title (no empty titles allowed)
- Same pattern as existing `TaskItem`

### Status Advancement
- Click on status badge advances to next status in transition chain
- Badge is disabled (cursor-default, opacity-70) when at terminal status
- Title attribute shows "Mark as {nextLabel}" or current label when terminal

### Drag to Reorder (orderable: true)
- Grip handle icon on left side of item
- @dnd-kit with PointerSensor + KeyboardSensor
- Dragging item has opacity 0.5
- Drop calculates midpoint position between neighbors
- Same DndContext/SortableContext pattern as existing TaskList

### View Switcher
- Persisted to localStorage key `{featureName}-view-preference`
- Default view from YAML `views.default.defaultView`
- Switch is instant (no loading state) -- data is already fetched
- View switcher placed between heading section and content, aligned right

### Pagination (table view only)
- Offset-based, default 20 items per page
- "Previous" / "Next" buttons, disabled at boundaries
- Current page highlighted
- Page count info: "Showing {start}-{end} of {total}"

### Infinite Scroll (list/card/cover views)
- Load more when user scrolls within 200px of bottom
- Loading indicator: spinner or "Loading more..." text below last item
- No "Load More" button -- automatic only

### Filter Controls (for declarative filtered views)
- Tab-style row above the list: `flex items-center gap-1 border-b border-border pb-2`
- Each tab: `px-3 py-1.5 text-sm font-semibold transition rounded-t`
- Active tab: `text-primary border-b-2 border-primary`
- Inactive tab: `text-muted-foreground hover:text-primary`
- "All" tab always first (shows unfiltered)
