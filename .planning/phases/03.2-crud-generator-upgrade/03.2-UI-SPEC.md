# Phase 03.2 — UI Design Contract (Premium)

> Visual and interaction contract for generated CRUD components. This phase produces Handlebars templates that output React components. Generated code must look like a premium data app (Frappe CRM, Glide Apps, Linear) -- not a basic admin panel. Every generated feature should be immediately demoable to a client.

**Design Direction:** Generous whitespace, subtle depth through shadows, smooth transitions, restrained semantic color, professional empty/loading states, and clear visual hierarchy. The goal is "designed" not "scaffolded."

**Glide Apps Inspiration:** Multiple view types with configurable density, image variants, and slide-in detail panels. Collection-level title bars with action buttons and integrated search. Every view type is YAML-configurable for size, density, and display style.

## Design System

| Property | Value |
|----------|-------|
| Tool | none (manual shadcn pattern -- CSS variables + Radix primitives + CVA) |
| Preset | not applicable (`components.json` does not exist; tokens defined in `src/index.css`) |
| Component library | Radix UI primitives (wrapped in `src/ui/`) |
| Icon library | lucide-react |
| Font | Inter Variable (body + UI), system-ui fallback |

**Font Upgrade:** Replace `system-ui, sans-serif` in `src/index.css` body rule with `"Inter Variable", "Inter", system-ui, sans-serif`. Inter Variable is the same font used by Frappe's Espresso design system and Linear. It provides optical sizing, tabular numbers for tables, and a professional feel without adding a second font family. Install via `@fontsource-variable/inter` (tree-shakeable, self-hosted, no Google Fonts external request).

```bash
npm install @fontsource-variable/inter
```

Add to `src/index.css` before the Tailwind import:
```css
@import "@fontsource-variable/inter";
```

Update body font-family:
```css
body {
  font-family: "Inter Variable", "Inter", system-ui, sans-serif;
}
```

## Spacing Scale

8-point grid base. All values multiples of 4px. Generous spacing is the single biggest visual upgrade from "scaffolded" to "designed."

| Token | Value | Tailwind | Usage in Generated Code |
|-------|-------|----------|------------------------|
| xs | 4px | `gap-1`, `p-1` | Dot-to-label gaps inside status badges, icon button internal padding |
| sm | 8px | `gap-2`, `p-2` | Inline form element gaps, badge internal padding, compact density list items |
| md | 12px | `gap-3`, `p-3` | List item internal padding (px-3 py-3), card field spacing |
| lg | 16px | `gap-4`, `p-4` | Card internal padding (compact size), form field vertical spacing, grid gaps |
| xl | 20px | `gap-5`, `p-5` | Card generous padding (default size), detail page field groups |
| 2xl | 24px | `gap-6`, `p-6` | Section-to-section spacing (heading-to-content), large card padding |
| 3xl | 32px | `gap-8`, `p-8` | Page-level vertical padding, major section breaks |
| 4xl | 48px | `gap-12`, `p-12` | Empty state vertical centering padding |
| 5xl | 64px | `gap-16`, `p-16` | Not used in current patterns -- reserved |

**Key upgrade from existing spec:** Item padding increased from `px-3 py-2` (cramped) to `px-4 py-3` (comfortable). Card padding increased from `p-4` to `p-5` (generous). Section spacing increased from `gap-4` to `gap-6`.

Exceptions:
- Touch targets: all interactive elements minimum 36px height (h-9). Status dot badges at 32px height (h-8) acceptable on desktop -- they contain text labels, not just icons.
- Drag handles: 16x16px icon (`h-4 w-4`) inside a 32px touch zone (p-2 around the icon).
- Horizontal scroll cards: snap points at 280-320px width with 16px gap.

| Element | Radius | Tailwind | Rationale |

| Image thumbnails | 6px | `rounded-md` | Inline images in tables, list items, and cards. |
| Slide-in panel | 8px left | `rounded-l-lg` | Right-anchored panel has radius on left edge only. |

## Typography

Inter Variable at two weights. Three size tiers with distinct roles. Body text at 14px (existing) is correct for data-dense UIs. Headings step up to create clear hierarchy.

| Role | Size | Weight | Line Height | Letter Spacing | Tailwind Classes |
|------|------|--------|-------------|---------------|-----------------|
| Body | 14px | 400 (regular) | 1.5 (21px) | normal | `text-sm` |
| Label / Caption | 12px | 500 (medium) | 1.5 (18px) | 0.01em | `text-xs font-medium tracking-wide` |
| Heading (page) | 20px | 600 (semibold) | 1.3 (26px) | -0.01em | `text-xl font-semibold tracking-tight` |
| Section heading | 14px | 600 (semibold) | 1.5 (21px) | normal | `text-sm font-semibold` |

**Key changes from existing spec:**
- Page heading reduced from 24px (`text-2xl`) to 20px (`text-xl`) -- tighter, more professional. Frappe CRM and Linear use compact headings.
- Labels use weight 500 (medium) instead of 600 (semibold) -- subtler emphasis, less visual noise.
- Section headings at body size (14px) with semibold weight -- hierarchy through weight, not size.
- Added tracking-tight on headings and tracking-wide on labels for professional typographic polish.

**Table-specific typography:**
- Column headers: `text-xs font-medium uppercase tracking-wider text-muted-foreground` (12px, uppercase, wide tracking -- Frappe/Linear pattern)
- Cell content: `text-sm` (14px, regular weight)
- Numeric cells: `tabular-nums` class for aligned columns (Inter Variable supports this natively)

**Group headers (table row grouping):**
- `text-xs font-semibold uppercase tracking-wider text-muted-foreground` -- same as column headers but semibold weight to distinguish from column labels.

## Color

The project uses ShadCN's HSL CSS variable system. Generated components MUST use semantic token classes, never raw hex/HSL values, except for enum badge semantic colors.

### Core Palette (60/30/10)

| Role | Token | Tailwind Class | Usage |
|------|-------|---------------|-------|
| Dominant (60%) | `--background` | `bg-background` | Page background, scrollable areas |
| Surface (secondary fill) | `--surface-secondary` (NEW) | `bg-surface-secondary` | Form areas, filter bar background, sidebar, section backgrounds, striped table rows |
| Card (30%) | `--card` | `bg-card` | Cards, list items, table rows, detail panels |
| Accent (10%) | `--primary` | `bg-primary`, `text-primary` | Primary CTA button, page heading, item title text |
| Destructive | `--destructive` | `text-destructive` | Delete button text and icon only |
| Muted | `--muted` | `bg-muted` | Disabled states, skeleton placeholders base, inactive view switcher |
| Border | `--border` | `border-border` | Card borders, input borders, section dividers, table grid lines |

**Accent reserved for:** Primary CTA button fill (`bg-primary text-primary-foreground`), page heading text (`text-primary`), item title text (`text-primary`), active view switcher segment (`bg-background` with shadow, inside `bg-muted` container), active filter tab underline, focus rings (`ring-ring`), link column text (`text-primary` with underline-on-hover). Never used for decorative backgrounds or large surface fills.

**Opacity scale (existing pattern, continue in generated code):**
| Opacity | Tailwind | Usage |
|---------|----------|-------|
| 60% | `text-primary/60` | Description text, secondary labels, page subtitle, emphasis field |
| 50% | `text-primary/50` | Placeholder text in empty states |
| 40% | `text-primary/40` | Drag handle inactive, unassigned icon |
| 20% | `text-primary/20` | Inactive toggle icons (e.g., low-priority flag), backdrop overlay |
| 5% | `hover:bg-primary/5` | Subtle hover backgrounds on interactive text elements |

### Enum Badge Semantic Colors

Status and enum badges use a dot indicator + text pattern (not icon + text). Colors are assigned round-robin from the palette below, based on declaration order in the YAML `values` array.

| Slot | Dot Color | Text Color | Usage Hint |
|------|-----------|------------|------------|
| 0 | `bg-gray-400` | `text-gray-600 dark:text-gray-400` | Default/neutral states (draft, pending) |
| 1 | `bg-blue-500` | `text-blue-600 dark:text-blue-400` | Active/in-progress states |
| 2 | `bg-green-500` | `text-green-600 dark:text-green-400` | Success/completed states |
| 3 | `bg-amber-500` | `text-amber-600 dark:text-amber-400` | Warning/on-hold states |
| 4 | `bg-red-500` | `text-red-600 dark:text-red-400` | Error/blocked/rejected states |
| 5 | `bg-purple-500` | `text-purple-600 dark:text-purple-400` | Special/custom states |
| 6 | `bg-pink-500` | `text-pink-600 dark:text-pink-400` | Overflow slot 1 |
| 7 | `bg-cyan-500` | `text-cyan-600 dark:text-cyan-400` | Overflow slot 2 |

**Badge markup pattern (dot indicator style):**
```html
<span class="inline-flex items-center gap-1.5 text-xs font-medium {textColorClass}">
  <span class="h-2 w-2 rounded-full {dotColorClass}" aria-hidden="true"></span>
  {label}
</span>
```

**Key change from existing spec:** Replaced `bg-{color}-100 text-{color}-700` pill badges with dot indicator + text-only style. This is the Linear/Frappe pattern -- cleaner, less visual noise, works better in tables and dense lists. No background color on the badge itself.

**Tag column pills (distinct from enum badges):**
Tag columns use the filled-pill style that enum badges replaced. This creates a clear visual distinction -- enum status = dot indicator, tag = colored pill.
```html
<span class="inline-flex items-center rounded-full bg-{color}-100 text-{color}-700 dark:bg-{color}-900/30 dark:text-{color}-400 px-2 py-0.5 text-xs font-medium">
  {tagLabel}
</span>
```

Color assignment: same palette slots as enum badges, assigned round-robin by declaration order in YAML `values` array.

**Override:** YAML `colors:` map on an enum field overrides auto-assignment. Example: `status: { colors: { active: 1, archived: 0 } }` assigns slot 1 (blue) to "active" and slot 0 (gray) to "archived".

## Shadows and Depth

Three shadow tiers. Shadows create the visual depth that distinguishes premium from flat admin UIs.

| Tier | CSS Variable | Tailwind (via theme) | Usage |
|------|-------------|---------------------|-------|
| Rest | `--shadow-card` | `shadow-card` | Cards, list items, table container -- subtle resting depth |
| Hover | `--shadow-card-hover` | `shadow-card-hover` | Cards on hover -- perceptible lift |
| Elevated | `--shadow-elevated` | `shadow-elevated` | Dropdowns, popovers, modals, slide-in panels -- clear layering |

**Card hover effect (MANDATORY on all cards):**
```
transition: box-shadow var(--transition-base) ease, transform var(--transition-base) ease;
&:hover { box-shadow: var(--shadow-card-hover); transform: translateY(-1px); }
```

Tailwind implementation:
```
shadow-card hover:shadow-card-hover hover:-translate-y-px transition-all duration-200 ease-out
```

## Transitions and Animation

All interactive elements have transitions. No instant state changes. Purpose-driven animation, not decorative.

| Category | Duration | Easing | Tailwind | Usage |
|----------|----------|--------|----------|-------|
| Micro-interaction | 150ms | ease-out | `duration-150 ease-out` | Button press feedback, toggle states, focus rings |
| State change | 200ms | ease-out | `duration-200 ease-out` | Card hover lift, row highlight, view switcher active segment |
| Layout shift | 300ms | ease-in-out | `duration-300 ease-in-out` | View type switching, expanding/collapsing sections, filter transitions, slide-in panel, group collapse/expand |

**Skeleton loading pulse:**
```css
@keyframes skeleton-pulse {
  0%, 100% { opacity: 0.4; }
  50% { opacity: 0.7; }
}
```
Tailwind: `animate-pulse` (built-in). Apply to `bg-muted rounded-md` placeholder elements.

**Slide-in panel animation:**
```css
/* Panel enters from right */
.panel-enter { transform: translateX(100%); }
.panel-enter-active { transform: translateX(0); transition: transform 300ms ease-out; }
.panel-exit { transform: translateX(0); }
.panel-exit-active { transform: translateX(100%); transition: transform 200ms ease-in; }
```

**prefers-reduced-motion:** All transitions and animations MUST be wrapped:
```css
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}
```

### Table View (`{PascalName}TableView.tsx`)

Supports two design variants, extended column types, column width sizing, row grouping, row click-through, and a page size selector.

**Design Variants (YAML `views.table.design`):**

| Variant | Description | Row Style |

**Column Types:**

| Type | Rendering | Tailwind Classes |
|------|-----------|-----------------|
| `string` | Plain text, first column gets `font-medium` | `text-sm text-primary` |
| `text` | Truncated long text | `text-sm text-primary truncate max-w-[300px]` |
| `number` | Right-aligned, tabular figures | `text-sm text-right tabular-nums` |

| `tag` | Colored pill badges | `bg-{color}-100 text-{color}-700 dark:bg-{color}-900/30 dark:text-{color}-400 rounded-full px-2 py-0.5 text-xs font-medium` (multiple tags wrap inline with `flex flex-wrap gap-1`) |

**Column Width Sizing (YAML per-column `width`):**

| Width Token | Pixels | Tailwind | Usage |

## Interaction Contracts

### Click-to-Edit Title
- Single click on title text enters edit mode (inline `<input>`)
- Enter saves, Escape cancels, blur saves
- Empty input on save reverts to original (no empty titles)
- Input has same font styling as display text (seamless transition)
- Subtle focus ring on edit input: `ring-2 ring-ring ring-offset-2`

### Status Advancement
- Click on dot-style badge advances to next status in transition chain
- Badge shows hover cursor (`cursor-pointer`) when actionable
- Badge is disabled (`cursor-default opacity-60`) at terminal status
- Title attribute: "Mark as {nextLabel}" or current label when terminal
- Status change has no confirmation (single click = immediate advancement)

### Card Click-Through
- Clicking anywhere on a card (except action buttons) navigates to detail page/panel/overlay
- Action buttons have `e.stopPropagation()` to prevent navigation
- Cursor: `cursor-pointer` on the card wrapper

### Row Click-Through (table)
- Clicking anywhere on a table row (except action buttons/inline buttons) opens detail view
- Action buttons and button-type columns have `e.stopPropagation()`
- Cursor: `cursor-pointer` on `<tr>` elements
- Keyboard: Enter on focused row opens detail

### Drag to Reorder (orderable: true)
- Grip handle icon on left side of item
- @dnd-kit with PointerSensor + KeyboardSensor
- Dragging item: `opacity-50`, subtle `shadow-elevated`
- Drop calculates midpoint position between neighbors
- DndContext/SortableContext wrapping pattern from existing TaskList

### View Switcher
- State persisted to `localStorage` key `{featureName}-view-preference`
- Default from YAML `views.default.defaultView`
- Switch is instant (data already fetched, just re-renders)
- Active segment has `bg-background shadow-sm` treatment (appears elevated from muted track)
- Placed in page title bar, right-aligned

### Detail Panel/Overlay
- Panel opens on row/card click when `views.detail.display: panel`
- Overlay opens on row/card click when `views.detail.display: overlay`
- Panel: Escape or backdrop click closes. Focus trapped inside panel.
- Overlay: Escape, backdrop click, or X button closes. Focus trapped inside dialog.
- URL updates with record ID as query param (`?detail={id}`) for shareable links
- Browser back navigates away from detail (panel/overlay closes)

### Edit Mode Toggle
- "Edit" button in action bar activates inline editing
- All read-only field values become their corresponding input types
- "Save" validates and submits, returns to read mode on success
- "Cancel" discards changes, returns to read mode
- Unsaved changes: if user clicks away or closes panel, show browser `beforeunload` prompt

### Search (collection header)
- Input debounces at 300ms
- Searches across title + description fields by default (configurable via YAML)
- Empty search clears filter
- Shows result count update in real-time

### Pagination (table view)
- Offset-based, default 20 items/page (configurable via page size selector)
- Page size selector: dropdown with 10 / 20 / 50 / 100 options
- Page size persisted to `localStorage` key `{featureName}-page-size`
- Previous/Next buttons, disabled at boundaries
- Active page number: `bg-primary text-primary-foreground rounded-md`
- Show max 5 page numbers with ellipsis
- Info text: "Showing {start}-{end} of {total}"

### Infinite Scroll (list/card/cover)
- IntersectionObserver triggers when sentinel element enters viewport
- Sentinel positioned after last item
- Loading state: 2 additional skeleton items below content
- No "Load More" button -- automatic

### Row Group Collapse/Expand
- Click group header row to toggle collapse
- Chevron rotates 90 degrees on toggle (duration-200 transition)
- Collapsed groups remember state in component state (not persisted)
- All groups expanded by default on page load

### Horizontal Card Scroll
- Native scroll with CSS snap points
- Scroll indicators (gradient fades) appear only when content overflows
- Left fade hidden when at scroll start, right fade hidden when at scroll end
- Scroll position detected via IntersectionObserver on first/last card

### Hover States Summary
| Element | Hover Effect |
|---------|-------------|
| Card | Shadow lift + translateY(-1px) |
| List item | Shadow lift + border lightens |
| Table row | `bg-muted/30` background |
| Group header row | `bg-muted/50` background |
| Action buttons | Opacity 0->100 (appear on parent hover) |
| Filter tab | `text-primary` color shift |
| View switcher segment | `text-primary` color shift |
| Status badge | `bg-primary/5` subtle background |
| Link text (and link columns) | `text-primary` + underline |
| Overlay card image | slight scale-up: `group-hover:scale-105 transition-transform duration-300` |

## YAML Configuration Reference

All YAML options that affect the UI, consolidated for generator template authors. Each option listed with its section, key, allowed values, and default.

### View-Level Configuration

```yaml
views:
  default:
    defaultView: list | card | table | cover    # Default: list
    enabledViews: [list, card, table, cover]     # Default: [list, card, table]

  table:
    design: minimal | striped                    # Default: minimal
    pageSize: 10 | 20 | 50 | 100                # Default: 20
    groupBy: <fieldName>                         # Default: null (no grouping)
    columns:
      - field: <fieldName>
        width: auto | xs | sm | md | lg | xl     # Default: auto
        type: string | text | number | boolean | enum | image | link | tag | button  # Inferred from field type
        sortable: true | false                   # Default: true for string/number/enum

  card:
    size: compact | default | large              # Default: default
    orientation: grid | horizontal               # Default: grid
    imageStyle: none | thumbnail | cover | overlay  # Default: none

  list:
    density: compact | comfortable               # Default: comfortable
    emphasis: <fieldName>                         # Default: null (no emphasis line)

  detail:
    display: page | panel | overlay              # Default: page
```

### Field-Level Configuration (affecting UI)

```yaml
fields:
  <fieldName>:
    type: string | text | number | boolean | enum | image | link | tag
    showOn: [list, card, table, detail, form]    # Default: all views
    hideOn: [list, card, table, detail, form]    # Default: none
    width: auto | xs | sm | md | lg | xl         # Table column width. Default: auto
    sortable: true | false                       # Table column sortable. Default: true
    values: [val1, val2, ...]                    # For enum/tag types
    colors:                                      # Override auto-assigned palette slots
      val1: 0                                    # Palette slot index
      val2: 1
```

### Search Configuration

```yaml
search: true | false                             # Default: false
searchFields: [title, description]               # Default: [title, description] when search: true
```

### Behavior Configuration (UI-relevant subset)

```yaml
orderable: true | false                          # Drag handles on list items. Default: false
assignable: true | false                         # Assignee avatar + dropdown. Default: false
auditTrail: true | false                         # Activity timeline on detail. Default: false
softDelete: true | false                         # "Show deleted" toggle. Default: false
immutable: true | false                          # Read-only form after commit. Default: false
```
