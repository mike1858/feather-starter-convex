## Decisions Made
- Use `@root.property` instead of relative `../../` paths for deeply nested Handlebars contexts (Plop's scope resolution is unreliable past 2 levels)
- Add dedicated helpers (`apiPath`, `errorsPath`) rather than fixing templates inline, for maintainability
- Generated schema uses `userId` field (not `creatorId`) to match wiring auto-generate pattern
- Compute `statusFieldKey` in yaml-resolver rather than hardcoding in templates

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Handlebars scope resolution failures in deeply nested test templates**
- **Found during:** Task 1 (generator pipeline run)
- **Issue:** `../../../name` resolved to undefined in templates nested inside `#each fields > #each this.transitions > #each this`
- **Fix:** Replaced all deep relative paths with `@root.name` / `@root.statusFieldKey`
- **Files modified:** templates/feature/mutations.test.ts.hbs
- **Committed in:** dd9697c

**2. [Rule 1 - Bug] Triple-stache collision between JSX braces and Handlebars expressions**
- **Found during:** Task 1 (typecheck of generated page component)
- **Issue:** `variant={{{#if ...}}` created `{{{` which Handlebars parsed as triple-stache (raw output)
- **Fix:** Added tilde whitespace control: `{ {{~#if~}} ... {{~/if~}} }` and `{{@key~}} }` pattern
- **Files modified:** templates/feature/page.tsx.hbs, templates/feature/item.tsx.hbs
- **Committed in:** dd9697c

**3. [Rule 1 - Bug] ifEq used as Handlebars subexpression but only implemented as block helper**
- **Found during:** Task 1 (title-bar template rendering)
- **Issue:** `{{#unless (ifEq a b)}}` failed with "options.inverse is not a function"
- **Fix:** Added `eq` helper returning boolean, replaced `(ifEq ...)` with `(eq ...)` in subexpressions
- **Files modified:** generators/utils/handlebars-helpers.js, templates/feature/page.tsx.hbs, test.tsx.hbs, title-bar.tsx.hbs
- **Committed in:** dd9697c

**4. [Rule 1 - Bug] Kebab-case names produce invalid JS identifiers in api/errors paths**
- **Found during:** Task 1 (typecheck on `api.test-gen.mutations`)
- **Issue:** `api.test-gen` and `ERRORS.test-gen` are invalid JS; need bracket notation
- **Fix:** Added `apiPath`/`errorsPath` helpers producing `api["test-gen"]` / `ERRORS["test-gen"]`
- **Files modified:** generators/utils/handlebars-helpers.js, all templates using api/ERRORS
- **Committed in:** dd9697c

**5. [Rule 1 - Bug] StatusBadge template only provided dot class, missing text class**
- **Found during:** Task 1 (typecheck on StatusBadge component)
- **Issue:** Color map objects had `{ dot: "..." }` but type required `{ dot, text }`
- **Fix:** Added `enumBadgeDot`/`enumBadgeText` separate helpers
- **Files modified:** generators/utils/handlebars-helpers.js, templates/feature/status-badge.tsx.hbs
- **Committed in:** dd9697c

**6. [Rule 1 - Bug] verticalListSortingStrategy typo in list-view template**
- **Found during:** Task 1 (typecheck)
- **Issue:** `sorticalListSortingStrategy` instead of `verticalListSortingStrategy`
- **Fix:** Fixed typo
- **Files modified:** templates/feature/list-view.tsx.hbs
- **Committed in:** dd9697c

**7. [Rule 1 - Bug] Mutations create inserts duplicate fields for enum+transition types**
- **Found during:** Task 1 (typecheck on mutations)
- **Issue:** First loop wrote all fields from args, then second loop wrote default for enum+transition fields
- **Fix:** Exclude enum+transition fields from first loop, only set defaults in second
- **Files modified:** templates/feature/mutations.ts.hbs
- **Committed in:** dd9697c

**8. [Rule 1 - Bug] creatorId vs userId field name mismatch**
- **Found during:** Task 1 (typecheck)
- **Issue:** Templates referenced `creatorId` but wiring generates `userId` field
- **Fix:** Changed to `userId` in queries and mutations templates
- **Files modified:** templates/feature/queries.ts.hbs, templates/feature/mutations.ts.hbs
- **Committed in:** dd9697c
