    ```typescript
    export function extractCustomRegions(fileContent: string): Map<string, string>
    ```
    - Scans for `// @custom-start {key}` ... `// @custom-end {key}` pairs
    - Returns a Map where key = region name, value = content between markers (trimmed of the marker lines themselves, but preserving internal whitespace)
    - Also supports `{/* @custom-start {key} */}` ... `{/* @custom-end {key} */}` for JSX files
    - Regex pattern: `/\/\/\s*@custom-start\s+(\S+)\s*\n([\s\S]*?)\/\/\s*@custom-end\s+\1/g`
    - JSX pattern: `/\{\/\*\s*@custom-start\s+(\S+)\s*\*\/\}\s*\n([\s\S]*?)\{\/\*\s*@custom-end\s+\1\s*\*\/\}/g`

    ```typescript
    export function injectCustomRegions(newContent: string, customRegions: Map<string, string>): string
    ```
    - Takes freshly generated content with empty `@custom-start`/`@custom-end` blocks
    - For each matching key in the Map, replaces the content between markers with the saved content
    - Returns the merged file content
    - If a custom region key exists in the Map but not in the new content, appends it at the end as a comment noting it was orphaned
    - If a custom region key exists in the new content but not in the Map, leaves it as-is (empty)

    **Part A: Handlebars helpers** — Create `generators/utils/handlebars-helpers.ts`:

    Export a `registerHelpers(plop: NodePlopAPI): void` function that registers these helpers:

    1. `zodType(field)` — Returns Zod type string for a field:
       - `string` → `z.string().max(${field.max || 200})`
       - `text` → `z.string().max(${field.max || 5000})`
       - `boolean` → `z.boolean().default(${field.default ?? false})`
       - `number` → `z.number()`
       - `enum` → `z.enum([${field.values.map(v => `"${v}"`).join(", ")}])`
       Wraps in `.optional()` if `field.required === false`

    2. `zodTypeExport(field)` — For enum fields, returns the named export:
       `export const ${UPPER_SNAKE}_VALUES = [${values}] as const;\nexport const ${camelCase} = z.enum(${UPPER_SNAKE}_VALUES);`
       For non-enum fields, returns empty string.

    3. `convexValidatorType(field)` — Returns Convex validator for schema.ts:
       - `string`/`text` → `v.string()`
       - `boolean` → `v.boolean()`
       - `number` → `v.number()`
       - `enum` → `zodToConvex(${camelCaseSchemaRef})`

    4. `formInputType(field)` — Returns the React component name:
       - `string` → `Input`
       - `text` → `textarea`
       - `boolean` → `Switch`
       - `number` → `Input type="number"`
       - `enum` → `Select`

    5. `ifEq(a, b, options)` — Equality conditional: `a === b ? options.fn(this) : options.inverse(this)`

    6. `ifIn(value, array, options)` — Array membership: `array.includes(value) ? options.fn(this) : options.inverse(this)`

    7. `enumBadgeClasses(field, value)` — Returns dot color + text color CSS classes for an enum value based on palette slot assignment:
       Palette (8 slots): gray-400/gray-600, blue-500/blue-600, green-500/green-600, amber-500/amber-600, red-500/red-600, purple-500/purple-600, pink-500/pink-600, cyan-500/cyan-600.
       Slot = index of value in field.values array % 8. If field.colors override exists, use that slot instead.

    8. `upperSnakeCase(str)` — Converts `camelCase` or `kebab-case` to `UPPER_SNAKE_CASE`

    9. `json(obj)` — `JSON.stringify(obj)`

    10. `showOnView(field, viewName)` — Returns true if field should show on given view. Checks field.showOn (if "all" or includes viewName) and field.hideOn (excludes if viewName in hideOn).
