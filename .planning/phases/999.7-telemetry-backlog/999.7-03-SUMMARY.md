## Decisions Made
- **errorDigests separate table**: Digests stored in dedicated table rather than reusing devErrors -- keeps captured errors clean for the /dev/errors dashboard; digests are aggregated summaries from client instances
- **Deploy-time cron schedule**: Convex cronJobs() is evaluated at deploy time; runtime schedule changes require redeployment. Daily at 9:00 UTC is the default (per D-10)
- **Dynamic import for Sentry**: `await import("@sentry/react")` only when VITE_SENTRY_DSN is set -- Sentry code is tree-shaken away when DSN absent, zero bundle cost for users who don't need it
- **Sentry enabled in PROD only**: `enabled: import.meta.env.PROD` prevents Sentry noise in development while still validating DSN configuration
- **Batch digested marking**: IDs batched in groups of 50 to respect Convex argument size limits
- **buildTopErrors exported**: Utility function exported for direct unit testing separate from action orchestration
