    Create `bin/lib/telemetry.ts` with these exports (per D-01, D-02, D-03, D-17, D-18):

    **TelemetryEvent interface:**
    ```typescript
    export interface TelemetryEvent {
      command: string;            // e.g., "generate", "add", "validate"
      feature?: string;           // e.g., "tasks", "contacts"
      bundle?: string;            // e.g., "project-management"
      success: boolean;
      durationMs: number;
      timestamp: string;          // ISO 8601
      error?: string;             // error message if success=false
      // schema-level fields (only when level >= "schema")
      fieldCount?: number;
      fieldTypes?: string[];
      relationships?: string[];
    }
    ```

    **isEnabled(projectRoot: string): boolean** — checks `FEATHER_TELEMETRY` env var first (if "false", return false), then reads feather.yaml `telemetry.enabled` field. Default: true.

    **appendEvent(projectRoot: string, event: TelemetryEvent): void** — uses NDJSON format (one JSON object per line, append-only via `fs.appendFileSync`). Creates `.feather/` directory if missing. Per D-01: writes to `.feather/telemetry.json`.

    **readEvents(filePath: string): TelemetryEvent[]** — reads NDJSON file, splits by newline, JSON.parse each line. Returns empty array if file doesn't exist. Ignores blank lines.

    **postRemote(projectRoot: string, event: TelemetryEvent): void** — reads `telemetry.remote` from feather.yaml. If set, POSTs event to that URL. Fire-and-forget: catches all errors silently. Uses native `fetch()`. Per D-02.

    **sanitizeEvent(event: TelemetryEvent): TelemetryEvent** — strips any field values that match sensitive patterns per D-18 (password, token, secret, key, authorization in field names). Deep-cleans the event object.

    **getTelemetryLevel(projectRoot: string): "basic" | "schema" | "full"** — reads `telemetry.level` from feather.yaml. Default: "basic". Per D-17, D-19.

    Then update `templates/schema/feather-yaml.schema.ts` to add telemetry and errorReporting sections to `projectYamlSchema` (per D-20):

    ```typescript
    telemetry: z.object({

      errorDigest: z.object({
        enabled: z.boolean().default(false),
        schedule: z.enum(["hourly", "daily", "weekly"]).default("daily"),
        endpoint: z.string().url(),
      }).optional(),
    }).optional(),
    errorReporting: z.object({
      sentry: z.object({
        dsn: z.string(),
      }).optional(),
    }).optional(),
    ```

    **StatsResult interface:**
    ```typescript
    export interface StatsResult {
      totalEvents: number;
      commandCounts: Record<string, number>;
      featureCounts: Record<string, number>;
      errorRate: number;            // 0-1
      failedCommands: number;
      mostGeneratedFeature: string | null;
      periodDays: number;
    }
    ```

    **statsAction(options: { days?: number }): StatsResult** — reads events from `.feather/telemetry.json` via `readEvents()`. Filters by `--days` (default 30). Aggregates:
    - `commandCounts`: count of events per `command` field
    - `featureCounts`: count of events per `feature` field (ignoring undefined)
    - `errorRate`: failedCommands / totalEvents
    - `mostGeneratedFeature`: feature with highest count

    **formatStats(result: StatsResult): string** — formats for terminal output. Table-like format showing:
    - Header: "Feather CLI Stats (last {N} days)"
    - Total commands: {N}
    - Error rate: {N}%
    - Section: "Commands" with counts
    - Section: "Features" with counts
    - Most generated: {feature} ({count})

    **statsCommand** — `new Command("stats").description("Show CLI usage statistics from local telemetry").option("--days <n>", "Time range in days", "30").action(...)`. The action prints `formatStats(statsAction(...))` to stdout.
