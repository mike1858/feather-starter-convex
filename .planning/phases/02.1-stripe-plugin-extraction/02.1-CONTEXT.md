# Phase 02.1: Stripe Plugin Extraction - Context

**Gathered:** 2026-03-10
**Status:** Ready for planning

<domain>
## Phase Boundary

Extract all Stripe/billing code from the core starter into an optional git-branch plugin (`plugin/billing`). After extraction, the core runs without Stripe ‚Äî no env vars, no SDK, no billing tables. The plugin branch adds everything back when merged. Also update 3 existing plugin branches (infra-ci, command-palette, admin-panel) for compatibility with the Stripe-free core.

</domain>

<decisions>
## Implementation Decisions

### Extraction Strategy
- Git branch plugin pattern (same as infra-ci, command-palette, admin-panel)
- Completely plan-agnostic core ‚Äî no concept of plans, subscriptions, or tiers
- Dashboard settings page stays, billing tab removed (plugin adds it back)
- All 3 existing plugin branches must be updated to work with Stripe-free core

### Onboarding Without Stripe
- Onboarding is just username setup ‚Äî no Stripe customer creation step
- Account deletion is just user record + auth accounts cleanup ‚Äî no subscription cancellation
- Post-onboarding lands on dashboard directly (no checkout redirect)

### Schema Decoupling
- Remove `plans` and `subscriptions` tables from core schema.ts
- Remove `customerId` field from users table
- Remove all billing constants (CURRENCIES, INTERVALS, PLANS) and validators from schema.ts
- Remove `src/shared/schemas/billing.ts` and billing Zod schemas from core
- Remove billing error group from errors.ts
- Simplify User type in types.ts ‚Äî just Doc<"users">, no subscription join
- Simplify getCurrentUser query ‚Äî return user profile only, no subscription/plan data
- Plugin branch adds all of this back

### Webhook/HTTP Separation
- Strip entire Stripe webhook handler from convex/http.ts (keep only auth routes)
- Strip or remove convex/init.ts (Stripe product seeding)
- Only billing-specific email templates move to plugin (subscriptionEmail.tsx)
- Email infrastructure (sendEmail, Resend integration) stays in core
- Remove STRIPE_SECRET_KEY and STRIPE_WEBHOOK_SECRET from convex/env.ts
- Remove `stripe` npm package from package.json

### Claude's Discretion
- Post-onboarding dashboard flow details (what shows on first visit without billing)
- How to structure the plugin branch diff for clean merges
- Test file handling ‚Äî which tests move to plugin vs stay in core
- Navigation component cleanup (removing billing-related nav items)
- vitest.config.ts cleanup (removing --dangerouslyIgnoreUnhandledErrors if Stripe was the cause)

</decisions>

<specifics>
## Specific Ideas

- Stripe blocks team in India from running the starter ‚Äî this is urgent/blocking
- The `--dangerouslyIgnoreUnhandledErrors` vitest flag was added specifically for pre-existing Stripe unhandled rejections ‚Äî may be removable after extraction
- Account deletion currently has a race condition with Stripe cancellation (CONCERNS.md) ‚Äî extraction eliminates this bug in core
- The `v8 ignore` directives in billing code (7 in stripe.ts, 2 in billing settings) all move to plugin ‚Äî core test coverage becomes more honest

</specifics>
